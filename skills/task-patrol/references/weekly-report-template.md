# 周报生成模板

## 触发条件

用户明确要求"生成周报"、"weekly report"或阶段 3.2 的交互式追问中选择周报选项时启用。

## 模板结构

```markdown
# Weekly Report - YYYY/MM/DD ~ YYYY/MM/DD

## 本周概览

| 指标 | 本周 | 上周 | 变化 |
|------|------|------|------|
| 总任务数 | N | M | ±X |
| 已完成 | A | B | ±Y |
| 新增任务 | C | D | ±Z |
| 活跃中任务数 | E | F | ±W |
| 长期滞留任务数 | G | H | ±V |

## 本周完成

1. [任务标题] (项目: xxx)
2. ...

## 当前聚焦 (P0)

1. [活跃状态名] [任务标题] (项目: xxx, 进展: ...)
2. ...

## 下周计划 (P1)

1. [状态名] [任务标题] (项目: xxx)
2. ...

## 风险与阻塞

- [风险类型] [具体描述] → [建议动作]

## 标签治理 (如有)

- [发现的问题] → [建议]

---
Generated by Task-Patrol on YYYY-MM-DD
```

## 生成规则

1. **时间范围**：默认最近 7 天（用户可指定起止日期，尊重用户团队的周起止习惯）
2. **数据来源**：基于本次 patrol 拉取的数据，不追溯历史（除非用户明确要求）
3. **已完成列表**：只列出状态为"已完成"且完成时间在报告周期内的任务
4. **跨项目模式**：在任务后标注 `(项目: projectId)`，并增加"项目分布"小节

## 项目分布小节（跨项目模式）

```markdown
## 项目分布

| 项目 | 本周完成 | 活跃中 | 待启动 | 风险等级 |
|------|----------|--------|--------|----------|
| project-a | 3 | 2 | 5 | 🟡 中 |
| project-b | 1 | 1 | 8 | 🔴 高 |

风险等级判断依据（根据实际数据分布动态评估）：
- 🟢 低：活跃中任务少，长期滞留占比低，整体可控
- 🟡 中：活跃中任务适中，或存在一定比例的长期滞留任务
- 🔴 高：活跃中任务过多，或长期滞留占比显著，需重点关注
```
